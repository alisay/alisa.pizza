What does the internet look like?

In early patent drawings of the internet, people didn't know how to represent it—is the internet a blob, a brain, a cloud, an explosion? People think the internet is mysterious (sometimes using metaphors like "the cloud")...

But the internet is really just a bunch of computers connected together.

The internet has no centre. This architectural decision gives the network its robustness. The engineers sought to build a “survivable communication network”… that is, they tried to design a communication network that easily handles individual failures. You could build highly reliable network out of unreliable components.

So to truly understand what you're looking at, it's important to know a bit of history. The internet isn't ethereal—the internet is physical.




In his book called Protocol, Alexander Galloway says:

“While many have debated the origins of the Internet, it’s clear that in many ways it was built to withstand nuclear attack. The Net was designed as a solution to the vulnerability of the military’s centralized system of command and control during the late 1950’s and beyond. For, the argument goes, if there are no central command centers, then there can be no central targets and overall damage is reduced.”

Computers date back to World War II
Scientist would use multiple computers to solve equations for developing atomic bombs.

But this still wasn’t the internet as we know it. The origin of the internet took some principles from these computers in WWII and created the first network called: ARPA in 1960

ARPANET by Lawrence G. Roberts — This was the first network of computers to communicate between different locations at a few universities in California.

More computers were added quickly to the ARPANET during the following years, and work proceeded on completing a functionally complete Host-to-Host protocol and other network software.

Transmission control protocol / Internet protocol was developed in the 1970 to allow different network systems to work together on one standardized system. We still use this protocol today.

The computers were able to communicate with one another using a packet switching mechanism.

Ray Tomlinson at BBN wrote the first basic email message send and read software, motivated by the need of the ARPANET developers for an easy coordination mechanism.

But looking for addresses for emails was quite tough. So the DNS // DOMAIN NAME SYSTEM was invented
A way to connect hosts yourname@whatever yourname@whatever.com / org / etc
